# 安全政策

## 支持版本

安全修复会提供给最新发布版本和当前 `main` 开发分支。

## 报告漏洞

请不要在公开 issue 中报告安全漏洞。

如果仓库启用了 GitHub 私有漏洞报告，请优先使用该渠道。如果没有私有报告入口，可以创建一个最小化的公开 issue，请求私下联系路径，但不要包含利用细节、日志、token、数据库凭据或敏感环境信息。

报告时请尽量包含：

- 受影响版本或 commit
- 问题的简洁描述
- 复现步骤或 proof of concept
- 预期影响
- 已知缓解方式

## 响应

这是一个小型开源项目，没有正式安全响应 SLA。维护者会尽力确认有效报告、调查问题，并在合适时发布修复或缓解方案。

## 范围

`uptime` 是一个轻量 `net/http` 中间件和状态 dashboard。

以下问题更适合报告到本项目：

- 敏感运行时信息或存储数据暴露
- dashboard 或 JSON API 中的不安全 HTTP 行为
- 会影响被包装服务的请求处理问题
- SQLite 或 PostgreSQL store 在并发使用下的正确性问题
- 告警 hook 去重失败导致通知风暴
- 对本包有实际影响的依赖漏洞

以下内容不属于本包范围：

- dashboard 认证或授权策略
- 反向代理配置
- 网络防火墙规则
- 通知服务账号安全
